# Changelog

Все значимые изменения в проекте TN_Doc будут документироваться в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [Unreleased] - v1.4.4

### Added
- 🚀 Система истории изменений полей паспорта качества
  - Отслеживание источника данных (ELIS, ручное редактирование, округление ИВК)
  - Визуальные индикаторы источников в UI (цветные значки)
  - Детальная история изменений в popup окне (до 10 записей на поле)
  - Автоматическая миграция из старого флага `ElisFilled`
  - Раздельная история для value/method/result/document полей
  - ⚠️ Требует включенного ELIS в конфигурации (`IsUsedElis = true`)
- 🚀 Document Editor (Vue 3 компонент) - в разработке
  - Редактирование параметров качества паспорта в браузере
  - Интеграция с ELIS для загрузки данных лаборатории
  - Интеграция с OPC для связи с устройствами
  - Автозаполнение зависимых параметров

### Changed
- Рефакторинг подсветки ELIS полей: логика на основе истории изменений
- Снято ограничение на количество записей истории полей (фронтенд)
- Улучшено логирование истории изменений полей

### Fixed
- Исправлена загрузка истории полей из initialValues для Passport
- Исправлено сохранение индикатора и подсветки ELIS при неизменённых значениях
- Исправлено отображение popup истории изменений: передача события для позиционирования
- Исправлен триггер открытия истории изменений: hover вместо click
- Исправлено создание истории изменений при загрузке из ELIS

## [1.4.3] - 2025-01-XX

### Added
- 🚀 Расширенный Configurator: настройки для измерительных приборов (СИ)
- 🚀 Configurator: настройки ELIS соединений
- 🚀 Configurator: настройки OPC соединений
- Централизация управления цветами через CSS переменные в `material3.css`

### Changed
- ⚠️ Все цвета UI теперь определены в `TN_Doc/wwwroot/css/material3.css`
- Заменены hardcoded HEX цвета на CSS переменные во всех стилях
- Обновлен docgeneral до версии 1.2.3

### Fixed
- 🐞 Исправлена печатная форма журнала регистрации СИ (совместимость с DataARM)

## [1.4.2] - 2025-10-02

### Removed
- ⚠️ Удален проект TN.Tools (устаревший функционал)

### Changed
- Упрощение определения путей до файлов
- Доработка KMH_MI2816 в связи с изменением структуры протокола в ИВК версии 7.12.14.3000
- Обновлен docgeneral до версии 1.2.2
- StatusBar: добавлены 4 состояния индикаторов (online, offline, ndv, warning)
- StatusBar: исправлено выравнивание индикаторов по левому краю
- StatusBar: увеличены размеры индикаторов устройств и сервисов
- StatusBar: удалена кнопка обновления статусов
- StatusBar: удален индикатор SignalR соединения
- StatusBar: убрано отображение времени последнего обновления

## [1.4.1] - 2025-XX-XX

### Changed
- ⚠️ Исправлены пути для логов:
  - Linux: `/opt/TN_Doc/logs`
  - Windows: `TN_Doc/logs`
- ⚠️ Выбор подписантов из локальных справочников пользователей для Отчетов и Журналов СИ
- ⚠️ Акт: время пробоотбора вместо времени смены
- ⚠️ Улучшение обработки ошибок и добавление логирования в функционал работы со Справочниками
- ⚠️ Построение документа in memory
- Рефакторинг главной страницы: упрощение интерфейса
- Рефакторинг классов управления принтером
- Логика подключения dll-библиотеки вынесена в docgeneral
- Перенос логики экспорта документа в соответствующий контроллер
- Логирование сообщений с клиента вынесено в отдельный контроллер
- Логирование событий при работе с документами (логирование библиотек)
- Корректировка лога заполнения формы редактирования ПК данными из ЕЛИС
- Обновлен docgeneral до версии 1.2.1

### Added
- Добавление обязательности заполнения поля 'Номер паспорта' во всех формах паспорта качества
- Справочник: при добавлении нового метода испытаний сразу делаем его активным
- Сортировка документов в КМХ и поверках по дате
- Сортировка Актов по дате
- Ручная корректировка (подстановки) вязкости для первого и последнего измерения Поверка 1974-2004, Поверка 1974-1995, Поверка 1974-1989

### Fixed
- 🐞 Доработка КМХ 3272 (bug-report с объекта)
- 🐞 Не верно заполняются смены Акта при заполнении паспортов "наоборот"
- Проверка запрещенных символов при автоматической подстановке данных в акт и паспорт
- Не показываем колонку Итого в Актах по умолчанию
- Кор. `21_Poverka3272_Release_version.frx` (bug-report с объекта)
- Кор. `16_Poverka3189_Release_version.frx`: Alpha_t1 -> Alpha_t_PU
- ЕЛИС: кор. заполнения колонки Результат-Текст в форме редактирования паспорта качества
- Отображение значения в заблокированной ячейке значения ХАЛ показателя качества формы редактирования паспорта качества
- Правка сообщений в логах: корректное отображение имени устройства
