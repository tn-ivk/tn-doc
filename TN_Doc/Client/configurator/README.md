# Конфигуратор настроек TN_Doc

Веб-интерфейс для управления конфигурацией приложения TN_Doc.

## Возможности

- **Вкладка "Общие"**:
  - Настройка пути экспорта документов
  - Включение/отключение функций безопасности
  - Конфигурация локального OPC-клиента (OPC DA/UA)
  - Настройка диагностики связи с ИВК (интервалы опроса, пороги отказов)

- **Вкладка "Устройства"**:
  - Список всех устройств с поиском
  - Множественный выбор устройств для группового редактирования
  - Индикация смешанного состояния при групповом редактировании
  - Настройка:
    - Включения/отключения устройства
    - Используемых документов
    - Подключения к БД
    - OPC настроек
    - Недопустимых символов

- **Вкладка "Документы"**:
  - Древовидная структура типов документов
  - JSON-редактор с подсветкой синтаксиса (Monaco Editor)
  - Панель редактирования конфигураций с предпросмотром
  - Валидация JSON перед сохранением

## Технологический стек

### Frontend
- Vue 3.4.21 + TypeScript
- PrimeVue 4.2+ (UI компоненты)
- Pinia (управление состоянием)
- Axios (HTTP клиент)
- Vite (сборка)

### Backend
- ASP.NET Core 8.0
- ConfiguratorController (API endpoints)
- ConfigurationService (бизнес-логика)
- Валидаторы для OPC и DB настроек

## Разработка

### Установка зависимостей
Из корня проекта `/TN_Doc/Client`:
```bash
npm install
```

### Запуск в режиме разработки
Из корня проекта `/TN_Doc/Client`:
```bash
npm run dev:configurator
```
Приложение будет доступно на http://localhost:5174

Или непосредственно в директории конфигуратора:
```bash
cd configurator
npm run dev
```

### Сборка для продакшена
Из корня проекта `/TN_Doc/Client`:
```bash
npm run build:configurator
```

Или для сборки всех проектов (statusbar + configurator):
```bash
npm run build:all
```

Результат сборки помещается в `/TN_Doc/wwwroot/configurator/`

## API Endpoints

- `GET /api/configurator/config` - получить конфигурацию
- `POST /api/configurator/config` - сохранить конфигурацию
- `POST /api/configurator/validate` - валидировать конфигурацию
- `GET /api/configurator/document-config?configPath=...` - загрузить конфигурацию документа
- `POST /api/configurator/document-config` - сохранить конфигурацию документа

## Использование

1. Запустите приложение TN_Doc
2. Откройте браузер: `http://localhost:38509/configurator`
3. Внесите необходимые изменения в конфигурацию
4. Нажмите "Применить" для сохранения

## Особенности

### Групповое редактирование
При выборе нескольких устройств:
- Изменения применяются ко всем выбранным устройствам
- Поля с разными значениями отмечаются предупреждением
- Placeholder показывает "— разные значения —"

### Валидация
- Обязательные поля проверяются перед сохранением
- OPC и DB настройки валидируются через отдельные валидаторы
- Ошибки отображаются в UI

### Логирование
- Все изменения конфигурации логируются через NLog
- Вычисляется diff изменений (что изменено, старое/новое значение)

### Предохранитель несохранённых изменений
- Браузер предупреждает о несохранённых изменениях при попытке закрыть страницу
- Кнопка "Сбросить" отменяет все изменения

### Диагностика связи с ИВК

Панель настроек диагностики позволяет управлять параметрами мониторинга подключений к устройствам:

| Параметр | Описание | Диапазон | По умолчанию |
|----------|----------|----------|-------------|
| Начальный интервал опроса | Интервал между проверками подключения | 1–3600 сек | 60 |
| Максимальный интервал опроса | Максимальный интервал при повторных отказах | 60–86400 сек | 3600 |
| Множитель интервала | Коэффициент увеличения интервала после отказа (шаг 0.1) | 1.1–10 | 2.0 |
| Порог сетевых ошибок | Количество ошибок для учёта отказа сети | 1–100 | 3 |
| Максимум попыток до блокировки | Попытки до перехода в HalfOpen состояние | 1–1000 | 24 |

Числовые поля диагностики используют только ручной ввод (без кнопок инкремента/декремента).

### Design Tokens

Конфигуратор использует унифицированную систему spacing через CSS-переменные:

```css
--space-1: 0.25rem;   /* 4px */
--space-2: 0.5rem;    /* 8px */
--space-3: 0.75rem;   /* 12px */
--space-4: 1rem;      /* 16px */
--space-5: 1.25rem;   /* 20px */
--space-6: 1.5rem;    /* 24px */
```

Все отступы, gap'ы и padding'и компонентов используют эти переменные вместо hardcoded значений.
