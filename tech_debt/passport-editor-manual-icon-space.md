# Паспорт качества: иконка ручного изменения не ставится при пробелах по краям

## Проблема
В форме редактирования паспорта качества (DocumentPassportEditor) при добавлении или удалении пробелов в начале/конце значений (номер паспорта, пункт приема сдачи нефти, номер аттестата, лаборатория и т.д.) иконка ручного изменения не появляется. Если добавить пробел внутри строки, иконка появляется корректно.

## Причина
При сравнении значений используется нормализация с `trim()`. Пробелы по краям удаляются, поэтому изменение считается отсутствующим или как «возврат к ELIS», и запись в историю ручных изменений не создается.

## План
1) Проверить точки сравнения (`FormFieldWithHistory`, `useFieldHistory`, `documentStore.updateField`) и зафиксировать правило: для текстовых полей пробелы по краям значимы; для чисел и дат сохранить текущую нормализацию.
2) Ввести общий helper, например `normalizeForCompare(fieldType, value)`: для text не использовать `trim`, для date/datetime использовать нормализацию по timestamp, для number оставить текущую числовую нормализацию.
3) Применить helper в `FormFieldWithHistory` (история/иконка) и в `documentStore.updateField` (проверка возврата к ELIS), определяя тип поля через конфигурацию.
4) Проверить вручную сценарии с пробелами слева/справа и внутри строки на ключевых полях — иконка ручного изменения должна появляться, ELIS-подсветка должна сбрасываться.
