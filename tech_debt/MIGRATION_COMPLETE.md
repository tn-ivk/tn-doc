## 🔧 Технические детали

### Паттерн реализации

Каждый документ следует единому паттерну:

```csharp
public class DocXXX : DocGeneral, IDocumentEditor
{
    #region IDocumentEditor Implementation

    public DocumentEditConfig GetEditConfig(int id)
    {
        // 1. Загрузка документа
        GetViewDoc(id);

        // 2. Построение списка полей
        var config = new DocumentEditConfig
        {
            DocId = id,
            DocType = "XXX",
            DeviceId = _deviceId,
            Fields = BuildFields(),
            InitialValues = ExtractInitialValues()
        };

        return config;
    }

    public bool SaveDocument(int id, Dictionary<string, object> values)
    {
        try
        {
            // 1. Загрузка документа
            GetViewDoc(id);

            // 2. Обновление AdditionalInfo
            var ad = GetAdditionalInfo();
            UpdateFromValues(ad, values);

            // 3. Сохранение в БД
            SaveToDatabase(id, ad);

            return true;
        }
        catch (Exception ex)
        {
            _logger.Error(ex, $"Ошибка сохранения: {ex.Message}");
            return false;
        }
    }

    #endregion
}
```

### Ключевые особенности

**1. Синхронные методы**
- НЕ используется Task/async/await
- Прямая реализация без промежуточных слоёв

**2. Композиция vs Наследование**
- BasePoverkaEditor используется через композицию
- Каждый документ сам решает, как реализовать методы

**3. Логирование**
- Trace: Вызовы методов
- Info: Успешные операции
- Error: Ошибки с полным стектрейсом

**4. Обратная совместимость**
- Старые методы GetEditDoc()/SaveDoc() сохранены
- Оба подхода работают параллельно
