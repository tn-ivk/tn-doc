# План усиления обработки путей (AppContext + CombineSafe)

## Цель
Устранить зависимость от рабочего каталога, централизовать сбор путей и повысить безопасность работы с файловой системой.

## Текущее состояние
- Переход на `AppContext.BaseDirectory` начат.
- Вспомогательные расширения `CombineSafe` используются частично.

## Доработки
1) Централизация
- Во всех местах заменить `Directory.GetCurrentDirectory()`/`Environment.CurrentDirectory` на `AppContext.BaseDirectory`.
- Использовать `CombineSafe` для склейки относительных путей, отбрасывать ведущие `..`.

2) Абсолютные пути
- Принимать абсолютные пути «как есть»; относительные — преобразовывать через базу приложения.
- В `LoadCfg<T>` уже добавлена проверка на абсолютность.

3) Проверки файлов
- Перед чтением: `File.Exists`; логировать отсутствие с полным путем.
- Для конфигураций: логировать размер, дату изменения (трассировка проблем деплоя).

4) Тесты
- Юнит‑тесты `CombineSafe` с «сложными» сегментами; проверка, что `..` не допускается наружу базового каталога.
- Интеграционные тесты: загрузка конфигов и шаблонов из подпапок.

## Этапы
- Этап 1: Ревизия и замена вызовов на `AppContext.BaseDirectory`.
- Этап 2: Внедрение `CombineSafe` повсеместно, тесты.

## Критерии готовности
- Нет прямых обращений к `Directory.GetCurrentDirectory`.
- Все пути независимы от CWD; тесты зелёные.
