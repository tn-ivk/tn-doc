## Подготовка к тестированию

### 1. Запуск приложения

```bash
# 1. Перейти в папку проекта
cd C:\dev\dotnet\ivk\tn_doc\TN_Doc

# 2. Установить окружение Development
$env:ASPNETCORE_ENVIRONMENT="Development"

# 3. Запустить приложение
dotnet run --urls="http://localhost:5000"
```

### 2. Сборка фронтенда

```bash
# В отдельном терминале
cd C:\dev\dotnet\ivk\tn_doc\TN_Doc\Client

# Пересобрать document-editor
npm run build:editor
```

### 3. Открыть редактор документа

1. Открыть браузер: `http://localhost:5000`
2. Выбрать устройство из списка
3. Выбрать тип документа "Passport" (Паспорт качества)
4. Открыть существующий документ или создать новый

### Шаги:

1. **Выбрать несколько документов** с разной степенью заполненности:
   - Минимальный (5-10 полей)
   - Средний (20-30 полей)
   - Максимальный (50+ полей)
2. **Для каждого документа**:
   - Открыть редактор
   - Сделать несколько изменений (3-5)
   - Сохранить документ
3. **Получить размер DataARM из БД**:
   ```sql
   SELECT
     IdDoc,
     CHAR_LENGTH(DataARM) AS size_chars,
     CHAR_LENGTH(DataARM) / 1024 AS size_kb
   FROM TableActAndPassportData
   WHERE IdDoc IN (123, 456, 789)
   ORDER BY size_kb DESC;
   ```

   | IdDoc | Количество полей | История (записей) | Размер DataARM (KB) | Размер истории (%) |

   | 123   | 10               | 5                 | ?                   | ?                 |
   | 456   | 30               | 15                | ?                   | ?                 |
   | 789   | 50               | 50                | ?                   | ?                 |

5. **Проверить лимит 50%**:
   - [ ] Размер истории не превышает 50% от размера DataARM
   - [ ] Если превышает - отметить для оптимизации
