# Промпт для E2E-тестирования Справочников TN_Doc

## Контекст

**Приложение:** TN_Doc - модуль документов для ИВК
**URL:** `http://localhost:5000/`
**Тестируемый функционал:** Модальное окно "Справочники"

---

## Структура интерфейса Справочников

### Главное меню (верхний уровень)
- **Персонал** — управление пользователями, группами, доверенностями
- **Методы испытаний** — настройка методов для паспортов качества

### Раздел "Персонал"

#### Подменю:
1. **Группы пользователей** — просмотр групп (только чтение)
2. **Доверенности** — CRUD операции
3. **Пользователи** — CRUD операции с фильтрацией по группам

#### Группы пользователей (ожидаемые):
- Представители испытательной лаборатории
- Представители сдающей стороны
- Представители принимающей стороны
- Представители ТНМ

#### Поля Пользователя:
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| Активен | checkbox | нет | Статус активности |
| Фамилия | textbox | да | Фамилия пользователя |
| Имя | textbox | да | Имя пользователя |
| Отчество | textbox | нет | Отчество пользователя |
| Организация | textbox | нет | Название организации |
| Должность | textbox | нет | Должность |
| Доверенность | combobox | нет | Выбор из справочника доверенностей |
| Сепаратор | checkbox | нет | Использовать разделитель при форматировании |
| Пробелы | checkbox | нет | Использовать пробелы при форматировании |
| Сокр. форма | checkbox | нет | Использовать сокращённую форму |

#### Поля Доверенности:
| Поле | Тип | Формат | Описание |
|------|-----|--------|----------|
| Активен | checkbox | — | Статус активности |
| Номер доверенности | textbox | строка | Номер документа |
| Дата доверенности | date | yyyy-MM-dd | Дата выдачи |

### Раздел "Методы испытаний"

#### Структура:
1. **Боковое меню** — список типов паспортов качества (10 типов)
2. **Селектор параметров** — выбор параметра для настройки методов
3. **Таблица методов** — CRUD операции

#### Типы паспортов качества:
- Паспорт для нефтепродукта
- Паспорт качества на экспорт
- Паспорт (ГОСТ Р50.2.40 Приложение Ж)
- Паспорт (ГОСТ Р50.2.40 Приложение И)
- Паспорт (МИ 3532-2015 Приложение 13)
- Паспорт (МИ 3532-2015 Приложение 14)
- Паспорт (МИ 3532-2015 Приложение 15)
- Паспорт (ТР ЕАЭС)
- 石油品质报告 (МИ 3532-2015 Приложение 15)
- Паспорт (ГОСТ Р50.2.40 Приложение Ж) (Экспорт)

#### Параметры (примеры):
- Температура нефти при условиях измерений объема, °C
- Давление нефти при условиях измерений объема, МПа
- Плотность нефти при 20 °C, кг/м³
- Массовая доля воды, %
- Массовая доля серы, %
- и другие (~19 параметров)

#### Поля Метода испытания:
| Поле | Тип | Описание |
|------|-----|----------|
| Активен | checkbox | Статус активности метода |
| Метод | textbox | Название метода (ГОСТ, ТУ и т.д.) |
| Контроль мин. значения | checkbox | Включить проверку минимума |
| Мин. значение | spinbutton | Минимальное допустимое значение |
| Сообщение | textbox | Сообщение при нарушении условия |
| Применять по умолчанию | checkbox | Использовать как метод по умолчанию |

## Сценарии тестирования

### 1. Управление пользователями

#### 1.1 Открытие справочников и проверка групп
```
1. Перейти на http://localhost:5000/
2. Нажать кнопку "Справочники"
3. Убедиться, что открылось модальное окно
4. Нажать пункт меню "Группы пользователей"
5. ПРОВЕРИТЬ: Отображаются 4 группы:
   - Представители испытательной лаборатории
   - Представители сдающей стороны
   - Представители принимающей стороны
   - Представители ТНМ
6. Сделать скриншот
```

#### 1.2 Добавление пользователя (все поля)
```
1. Перейти в "Пользователи"
2. Выбрать группу "Представители испытательной лаборатории"
3. Нажать кнопку "Добавить"
4. Заполнить ВСЕ поля:
   - Активен: включить
   - Фамилия: "Тестов"
   - Имя: "Тест"
   - Отчество: "Тестович"
   - Организация: "ООО ТестОрг"
   - Должность: "Инженер"
   - Доверенность: выбрать первую из списка
   - Сепаратор: оставить включенным
   - Пробелы: оставить включенным
   - Сокр. форма: оставить включенным
5. Нажать кнопку подтверждения (галочка)
6. ПРОВЕРИТЬ: Пользователь появился в таблице
7. Сделать скриншот
```

#### 1.3 Редактирование пользователя (изменить ВСЕ поля)
```
1. Найти созданного пользователя в таблице
2. Нажать кнопку редактирования (иконка замка/карандаша)
3. Изменить ВСЕ поля:
   - Активен: выключить, затем включить
   - Фамилия: "Тестов" → "Тестов-Измененный"
   - Имя: "Тест" → "Иван"
   - Отчество: "Тестович" → "Иванович"
   - Организация: "ООО ТестОрг" → "АО НоваяОрг"
   - Должность: "Инженер" → "Старший инженер"
   - Доверенность: выбрать другую
   - Сепаратор: переключить (снять)
   - Пробелы: переключить (снять)
   - Сокр. форма: переключить (снять)
4. Нажать кнопку подтверждения
5. ПРОВЕРИТЬ: Все изменения отображаются в таблице
6. Сделать скриншот
```

#### 1.4 Переключение между группами
```
1. Выбрать группу "Представители сдающей стороны"
2. ПРОВЕРИТЬ: Таблица обновилась (может быть пустая или с другими пользователями)
3. Выбрать группу "Представители принимающей стороны"
4. ПРОВЕРИТЬ: Таблица обновилась
5. Выбрать группу "Представители ТНМ"
6. ПРОВЕРИТЬ: Таблица обновилась
7. Вернуться к "Представители испытательной лаборатории"
8. ПРОВЕРИТЬ: Тестовый пользователь отображается
```

#### 1.5 Удаление пользователя
```
1. Найти тестового пользователя
2. Нажать кнопку удаления (корзина)
3. ПРОВЕРИТЬ: Пользователь удалён из таблицы
4. Сделать скриншот
```

### 2. Управление доверенностями

#### 2.1 Просмотр доверенностей
```
1. Перейти в "Доверенности"
2. ПРОВЕРИТЬ: Отображается таблица с колонками:
   - Активен
   - Номер доверенности
   - Дата доверенности
   - Действия
3. Сделать скриншот
```

#### 2.2 Добавление доверенности
```
1. Нажать кнопку "Добавить"
2. Заполнить поля:
   - Активен: включить
   - Номер: "ТСТ-001/2026"
   - Дата: "2026-01-21"
3. Подтвердить
4. ПРОВЕРИТЬ: Доверенность появилась в таблице
5. Сделать скриншот
```

#### 2.3 Редактирование доверенности
```
1. Нажать редактирование созданной доверенности
2. Изменить:
   - Номер: "ТСТ-001/2026" → "ТСТ-002/2026"
   - Дата: изменить на другую
3. Подтвердить
4. ПРОВЕРИТЬ: Изменения отображаются
```

#### 2.4 Удаление доверенности
```
1. Нажать удаление тестовой доверенности
2. ПРОВЕРИТЬ: Доверенность удалена из таблицы
```

### 3. Методы испытаний

#### 3.1 Навигация по паспортам качества
```
1. Перейти в "Методы испытаний"
2. ПРОВЕРИТЬ: Отображается список паспортов в боковом меню (10 типов)
3. Выбрать "Паспорт для нефтепродукта"
4. ПРОВЕРИТЬ: Отображается селектор параметров и таблица методов
5. Выбрать другой паспорт "Паспорт качества на экспорт"
6. ПРОВЕРИТЬ: Данные обновились
7. Сделать скриншот
```

#### 3.2 Переключение параметров
```
1. Выбрать параметр "Температура нефти при условиях измерений объема, °C"
2. ПРОВЕРИТЬ: Таблица методов обновилась
3. Выбрать параметр "Массовая доля воды, %"
4. ПРОВЕРИТЬ: Таблица методов обновилась
```

#### 3.3 Добавление метода испытания
```
1. Нажать "Добавить"
2. Заполнить все поля:
   - Активен: включить
   - Метод: "ГОСТ Р 12345-2026 Метод Т"
   - Контроль мин. значения: включить
   - Мин. значение: "0.5"
   - Сообщение: "Значение ниже допустимого"
   - Применять по умолчанию: включить
3. Подтвердить
4. ПРОВЕРИТЬ: Метод появился в таблице
5. Сделать скриншот
```

#### 3.4 Редактирование метода
```
1. Редактировать созданный метод
2. Изменить все поля:
   - Метод: добавить "v2" в конец
   - Мин. значение: изменить на "1.0"
   - Сообщение: изменить текст
   - Переключить чекбоксы
3. Подтвердить
4. ПРОВЕРИТЬ: Изменения сохранены
```

#### 3.5 Удаление метода
```
1. Удалить тестовый метод
2. ПРОВЕРИТЬ: Метод удалён из таблицы
```

### 4. Валидация

#### 4.1 Пустые обязательные поля (Пользователь)
```
1. Добавить нового пользователя
2. Оставить поле "Фамилия" пустым
3. Попытаться сохранить
4. ПРОВЕРИТЬ: Отображается ошибка валидации или поле подсвечивается
5. Сделать скриншот
```

#### 4.2 Недопустимые символы
```
1. Ввести в поле Фамилия: "Тест<>?!@#$%"
2. ПРОВЕРИТЬ: Ошибка валидации ИЛИ символы не вводятся
```

#### 4.3 Некорректное число (Методы)
```
1. Перейти в Методы испытаний
2. Редактировать метод
3. Ввести в "Мин. значение": "abc"
4. ПРОВЕРИТЬ: Ошибка валидации или ввод не принимается
```

### 5. Обнаружение изменений

#### 5.1 Закрытие без сохранения
```
1. Открыть справочники
2. Внести изменение (например, изменить имя пользователя)
3. Нажать кнопку закрытия окна (крестик)
4. ПРОВЕРИТЬ: Появился диалог "Сохранить изменения?" или изменения сохраняются автоматически
5. Сделать скриншот
```

#### 5.2 Глобальное сохранение
```
1. Внести несколько изменений в разных разделах
2. Нажать кнопку "Сохранить" (если есть)
3. Закрыть и открыть снова
4. ПРОВЕРИТЬ: Все изменения сохранились
```

### 6. Связи между сущностями

#### 6.1 Привязка доверенности к пользователю
```
1. Создать доверенность "ТСТ-СВЯЗЬ/2026"
2. Перейти к пользователям
3. Редактировать пользователя
4. Выбрать созданную доверенность из списка
5. Сохранить
6. ПРОВЕРИТЬ: Доверенность отображается в ячейке пользователя
```

#### 6.2 Проверка списка доверенностей в селекторе
```
1. Добавить новую доверенность
2. Перейти к редактированию пользователя
3. ПРОВЕРИТЬ: Новая доверенность появилась в выпадающем списке
```

### 7. Расширенные сценарии

#### 7.1 Множественное добавление
```
1. Добавить 3 пользователей подряд
2. ПРОВЕРИТЬ: Все 3 отображаются в таблице
3. Удалить всех тестовых пользователей
```

#### 7.2 Добавление пользователей в разные группы
```
1. Добавить пользователя в группу "Представители сдающей стороны"
2. Переключиться на группу "Представители принимающей стороны"
3. Добавить пользователя
4. Вернуться к первой группе
5. ПРОВЕРИТЬ: Пользователь из первой группы на месте
6. Переключиться ко второй группе
7. ПРОВЕРИТЬ: Пользователь из второй группы на месте
```

#### 7.3 Методы для разных паспортов
```
1. Выбрать паспорт "Паспорт для нефтепродукта", параметр "Температура..."
2. Добавить метод
3. Выбрать другой паспорт
4. Добавить метод для того же параметра
5. ПРОВЕРИТЬ: Методы независимы для разных паспортов
```

## Инструкция по запуску

### Предварительные требования
1. Запущен TN_Doc на `http://localhost:5000/`
2. Claude Code с Playwright MCP

### Запуск одного сценария
```
Выполни тестовый сценарий "1.2 Добавление пользователя" из файла
docs/docs/tn_doc/tech_debt/dictionaries-e2e-testing-prompt.md
используя Playwright MCP. Делай скриншоты после каждого шага.
```

### Запуск всех сценариев
```
Выполни все тестовые сценарии из файла
docs/docs/tn_doc/tech_debt/dictionaries-e2e-testing-prompt.md
по порядку, делая скриншоты и отмечая результаты (PASS/FAIL).
В конце создай отчёт о тестировании.
```

## Ожидаемые результаты

### Критерии успеха:
- [ ] Все CRUD операции для пользователей работают
- [ ] Все CRUD операции для доверенностей работают
- [ ] Все CRUD операции для методов испытаний работают
- [ ] Переключение между группами пользователей работает
- [ ] Переключение между паспортами качества работает
- [ ] Переключение между параметрами работает
- [ ] Валидация полей работает
- [ ] Связи между сущностями работают корректно
- [ ] Нет ошибок в консоли браузера (кроме SignalR если сервис не запущен)

### Известные особенности:
- SignalR ошибки (TN_MessagingService не запущен) — не критично для тестирования справочников
- Inline-редактирование (без модальных форм)
- Кнопки без текста (иконки) — определять по позиции и контексту
