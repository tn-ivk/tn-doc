# План внедрения OpenTelemetry (трейсинг/метрики/логи)

## Цель
Собрать трассы, метрики и логи для ключевых операций (загрузка DLL, рендер отчётов, экспорт, чтение конфигов).

## Доработки
1) Трейсинг
- Добавить `OpenTelemetry` SDK; источники активностей для контроллеров/DocModuleLoader/FastReport.
- Экспорт в OTLP (конфигурируемый endpoint).

2) Метрики
- Счётчики ошибок, гистограммы длительности операций, количество рендеров/экспортов.

3) Логи
- Увязка логов с трассами (traceId/spanId) через `ILogger` provider.

4) Тесты
- Юнит‑тест провайдера метрик (in‑memory), smoke‑тест трасс при запросе контроллера.

## Этапы
- Этап 1: базовый SDK и трассы контроллеров.
- Этап 2: метрики рендеринга/экспорта.
- Этап 3: связывание логов с трассами.

## Критерии готовности
- Трассы/метрики видны в целевой системе (Jaeger/Tempo/OTLP‑коллектор).
