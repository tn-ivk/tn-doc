## 🎯 Стратегия миграции

### Принципы приоритизации:

1. **От простого к сложному** - начинаем с документов, имеющих только текстовые поля
2. **Группировка по похожести** - мигрируем серии документов вместе (3265, 3312, 1974)
3. **Переиспользование кода** - создаем базовые классы для похожих документов
4. **Инкрементальное тестирование** - тестируем каждую группу перед переходом к следующей

### Критерии сложности:

**Уровень 1 (Простой):** Только текстовые поля в AdditionalInfo
**Уровень 2 (Средний):** Текстовые поля + select для операторов
**Уровень 3 (Сложный):** Таблицы измерений, сложная валидация, вычисляемые поля
**Уровень 4 (Очень сложный):** ELIS интеграция, динамические таблицы, сложная бизнес-логика

## 📝 Примечания

### Риски:

1. **Сложность некоторых документов** - может потребоваться больше времени
2. **ELIS интеграция** - может потребовать дополнительной работы
3. **Специфические бизнес-правила** - нужно тщательно тестировать
4. **Обратная совместимость** - старый HTML подход должен продолжать работать

### Возможности оптимизации:

1. **Параллельная разработка** - простые документы можно мигрировать одновременно
2. **Автоматизация** - создать скрипты генерации базового кода
3. **Шаблоны** - использовать шаблоны для однотипных документов
4. **Batch тестирование** - тестировать группы похожих документов вместе
