## 🚀 Пошаговое тестирование

### Шаг 1: Сборка Vue Document Editor

```bash
# Перейти в директорию Client
cd /mnt/c/dev/dotnet/ivk/tn_doc/TN_Doc/Client

# Собрать Document Editor для production
npm run build:editor
```

**Ожидаемый результат:**
- В консоли должно появиться сообщение об успешной сборке
- Файлы будут созданы в `TN_Doc/wwwroot/document-editor/`

**Проверка:**
```bash
ls -la /mnt/c/dev/dotnet/ivk/tn_doc/TN_Doc/wwwroot/document-editor/
```

Должны быть файлы:
- `index.html`
- `assets/` директория с JS и CSS файлами

---

### Шаг 2: Включить флаг UseVueDocumentEditor

Откройте файл `TN_Doc/Cfg/CfgApp.json`:

```bash
# Найти строку UseVueDocumentEditor
grep -n "UseVueDocumentEditor" /mnt/c/dev/dotnet/ivk/tn_doc/TN_Doc/Cfg/CfgApp.json
```

Измените значение на `true`:

```json
{
  "UseVueDocumentEditor": true
}
```

**Важно:** Сохраните файл!

---

### Шаг 3: Запустить backend

```bash
cd /mnt/c/dev/dotnet/ivk/tn_doc/TN_Doc
```

**Оставьте терминал открытым!**

---

### Шаг 4: Открыть приложение в браузере

1. Откройте браузер (рекомендуется Chrome или Firefox)
2. Перейдите по адресу: `http://localhost:38509`
3. Откройте **DevTools** (F12 или Ctrl+Shift+I)
4. Перейдите во вкладку **Console**

**Что проверить:**
- ✅ Главная страница загрузилась без ошибок
- ✅ В консоли нет красных ошибок
- ✅ Статус-бар внизу страницы отображается

---

### Шаг 5: Выбрать устройство и документ Report

1. **Выберите устройство** из выпадающего списка "Устройство"
   - Например: "ИВК-1"

2. **Выберите документ** "Отчеты" (Report) из выпадающего списка "Документ"

3. **Установите период дат:**
   - Дата начала: выберите дату (например, месяц назад)
   - Дата окончания: выберите сегодняшнюю дату

4. **Нажмите кнопку "Получить данные"**

**Ожидаемый результат:**
- В таблице слева появится список отчетов
- Каждый отчет должен иметь дату и возможно номер

**Проблема?**
- Если список пустой, выберите другой период дат
- Проверьте, что в БД есть данные для выбранного устройства

---

### Шаг 6: Выбрать Report и открыть редактор

1. **Кликните на любой отчет в списке** (строка в таблице)
   - Строка должна выделиться

2. **Нажмите кнопку "Редактирование"** в правом верхнем углу

**Что проверить в DevTools Console:**

Должны появиться логи:
```
Загружается Vue Document Editor: /document-editor/edit/0/Report/123
```

**Ожидаемое поведение:**
- Панель с кнопками "Форма документа", "Экспорт", "Печать" скрывается
- Панель с кнопкой "Сохранить" появляется
- В iframe начинает загружаться Vue SPA

---

### Шаг 7: Проверить загрузку Vue SPA

**В DevTools:**

1. Перейдите во вкладку **Network**
2. Проверьте запросы:
   - `GET /document-editor/edit/0/Report/123` - должен вернуть HTML
   - `GET /api/documents/0/Report/edit/123` - должен вернуть JSON с конфигурацией

3. Перейдите во вкладку **Console**
   - Не должно быть красных ошибок
   - Могут быть информационные логи от Vue

**Визуально:**
- В iframe должна загрузиться форма Vue редактора
- Заголовок: "Редактирование отчета"
- Таблица с полями:
  - Сдал (ФИО) 1
  - Сдал (ФИО) 2
  - Принял (ФИО) 1
  - Принял (ФИО) 2
- Кнопка "Сохранить" внизу

**Если Vue SPA не загружается:**

Проверьте в Console:
```javascript
// В консоли браузера выполните:
$('.FR').attr('src')
```

Должно показать URL вида: `/document-editor/edit/0/Report/123`

---

### Шаг 8: Проверить отображение данных

**Что проверить:**

1. **Поля формы заполнены данными из БД:**
   - Выпадающие списки показывают текущих пользователей
   - Если данные пустые, выпадающие списки показывают "(Не выбрано)" или первый элемент

2. **Поля редактируемые:**
   - Можно кликнуть на выпадающий список
   - Можно выбрать другого пользователя

**Проблема: данные не загружаются?**

Проверьте в DevTools Network:
```

```

```json

  "docId": 123,
  "deviceId": 0,
  "docType": "Report",
  "title": "Отчет ...",
  "fields": [
    {
      "name": "IdUser_Sdal_1",
      "label": "Сдал (ФИО) 1",
      "type": "select",
      "value": "5",
      "options": [...]
    },
    ...
  ]
}
```

### Шаг 9: Изменить значения полей

2. **Выберите другого пользователя** из списка

### Шаг 10: Сохранить документ

1. **Нажмите кнопку "Сохранить"** внизу формы

**Ожидаемое поведение:**

В DevTools Network должен появиться запрос:
```
POST /api/documents/0/Report/save/123
```

**Тело запроса (Request Payload):**
```json
{
  "IdUser_Sdal_1": "5",
  "IdUser_Sdal_2": "3",
  "IdUser_Prinal_1": "7",
  "IdUser_Prinal_2": "8"
}
```

**Ответ (Response):**
```json
{
  "success": true,
  "message": "Документ успешно сохранен"
}
```

**Визуально:**
- Должно появиться сообщение "Документ успешно сохранен"
- (Пока используется alert, в будущем будет Toast)

## 🐛 Возможные проблемы и решения

### Проблема 1: Vue SPA не загружается

**Симптомы:**
- Iframe пустой или показывает 404

**Решение:**
```bash
# Проверьте, что файлы собраны
ls -la /mnt/c/dev/dotnet/ivk/tn_doc/TN_Doc/wwwroot/document-editor/

# Пересоберите Vue приложение
cd TN_Doc/Client
npm run build:editor
```

### Проблема 2: API возвращает ошибку 500

**Симптомы:**
- В Network видно `GET /api/documents/.../edit/... - 500 Internal Server Error`

**Решение:**
- Проверьте логи backend в консоли
- Проверьте, что DocReport реализует IDocumentEditor
- Проверьте, что БД доступна

### Проблема 3: Данные не сохраняются

**Симптомы:**
- POST запрос возвращает success, но данные в БД не меняются

**Решение:**
- Проверьте логи backend
- Проверьте реализацию DocumentEditController.SaveDocument
- Проверьте права доступа к БД

### Проблема 4: Старый редактор загружается вместо Vue

**Симптомы:**
- Всегда загружается Blob URL вместо Vue SPA

**Решение:**
- Проверьте, что флаг `UseVueDocumentEditor = true` в CfgApp.json
- Перезапустите backend
- Проверьте в HomeController, что условие срабатывает:
  ```csharp
  if (_cfgApp.UseVueDocumentEditor && IsDocumentSupportedInVueEditor(IdDoc))
  ```

### Проблема 5: CORS ошибки

**Симптомы:**
- В консоли: "Access to XMLHttpRequest ... has been blocked by CORS policy"

**Решение:**
- Проверьте настройки CORS в Startup.cs
- Убедитесь, что Vue приложение запущено на том же домене
