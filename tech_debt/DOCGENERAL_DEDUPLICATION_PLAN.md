# DOCGENERAL_DEDUPLICATION_PLAN

## Контекст
- Цель: уменьшить дублирование кода между библиотеками в `tn.docgeneral` без изменения поведения документов.
- Фокус: поэтапный рефакторинг с контролем регрессий и сохранением уникальной логики отдельных модулей.
- Область: все проекты `tn.docgeneral` (документные модули, shared-библиотеки, утилиты и тесты).

## Этап 1. Инвентаризация и baseline
### Задача 1.1: Актуализировать список библиотек
- Подзадача 1.1.1: зафиксировать все `.csproj` внутри `tn.docgeneral`.
- Подзадача 1.1.2: отдельно отметить каталоги без `.csproj` (`ActProducer`, `ActRoute`).
- Подзадача 1.1.3: разделить на группы: `Document modules`, `Shared`, `Utils`, `Tests`.

### Задача 1.2: Зафиксировать исходные метрики
- Подзадача 1.2.1: собрать по каждому проекту количество `.cs` файлов и LOC.
- Подзадача 1.2.2: зафиксировать baseline по уровням схожести модулей.
- Подзадача 1.2.3: определить топ-приоритетные кластеры для первой волны.

### Критерий завершения этапа
- Полный реестр и baseline-метрики задокументированы в `CLAUDE.md` и/или отдельном tech debt документе.
