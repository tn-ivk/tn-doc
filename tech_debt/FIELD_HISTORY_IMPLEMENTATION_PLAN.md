# План внедрения истории изменений полей паспорта качества

## Описание задачи
Нужно реализовать сквозную историю изменений для всех полей редактора паспорта качества TN_Doc (AdditionalInfo, параметры качества, методы испытаний). Для каждого controlId требуется фиксировать источник правки (Manual / IVK / ELIS / Unknown), автора, дату, предыдущее и новое значения, комментарий. История хранится в `DataARM.FieldHistoryMap` (до 10 последних записей на поле) и передаётся между фронтендом и бэкендом через обновлённый контракт `EditData.History`. UI обязан показывать индикатор источника и popup с подробной историей (триггер: hover) как для значений параметров, так и для выбора метода испытаний. Миграция старых данных: однократное преобразование ElisFilled → FieldHistoryMap при загрузке документа в GetEditConfig().

## Принятые решения

### 1. FieldHistoryEntry.NewValue - УБРАНО ✅

**Решение:** Поле `NewValue` избыточно и удалено из модели. Используется только:
- `Value` - текущее (новое) значение после изменения
- `PreviousValue` - предыдущее значение (для отката/сравнения)

### 2. Компактная сериализация JSON - ОТЛОЖЕНА ⏸️

**Решение:** Реализуем с нормальными именами полей (Source, ModifiedAt, ModifiedBy, Value).
Компактную сериализацию (`s/t/a/v`) добавим в этапе 8 только если замеры покажут критичный размер DataARM.
