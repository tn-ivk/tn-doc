# План внедрения истории изменений полей паспорта качества

## Описание задачи
Нужно реализовать сквозную историю изменений для всех полей редактора паспорта качества TN_Doc (AdditionalInfo, параметры качества, методы испытаний). Для каждого controlId требуется фиксировать источник правки (Manual / IVK / ELIS / Unknown), автора, дату, предыдущее и новое значения, комментарий. История хранится в `DataARM.FieldHistoryMap` (до 10 последних записей на поле) и передаётся между фронтендом и бэкендом через обновлённый контракт `EditData.History`. UI обязан показывать индикатор источника и popup с подробной историей как для значений параметров, так и для выбора метода испытаний. Миграции старых данных не выполняются: новые записи формируются только при последующих изменениях.
