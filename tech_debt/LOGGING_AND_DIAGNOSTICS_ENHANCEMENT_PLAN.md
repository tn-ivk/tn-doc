# План улучшения логирования и диагностики

## Цель
Повысить наблюдаемость приложения: точные логи, метрики по ключевым операциям, единый формат.

## Доработки
1) Стандартизация логов
- Шаблоны сообщений: контекст (deviceId, idDoc, id, protocolNumber, RequestId).
- Уровни: Trace (детали), Debug (ключевые шаги), Error (исключения с стеками), Warn (деградированные пути).

2) Корреляция
- Прокидывать `RequestId` (или CorrelationId) во все логи контроллеров и сервисов.

3) Метрики
- Таймеры: рендер отчёта, экспорт файла, загрузка Doc DLL, чтение конфигов.
- Счётчики ошибок по типам исключений.
- Экспозиция (опц.) в Prometheus/Pushgateway.

4) Логи файловой подсистемы
- Размер и mtime конфигов/шаблонов при чтении; предупреждения при рассинхронизации.

5) Тесты
- Проверка формата логов (snapshot‑тесты), подсчёт метрик в тестовом провайдере.

## Этапы
- Этап 1: Стандарты сообщений и RequestId.
- Этап 2: Таймеры и счётчики.
- Этап 3: Тесты формата и подсчёта.

## Критерии готовности
- Единый формат логов, наличие ключевых полей.
- Метрики по критическим операциям доступны и валидны.
