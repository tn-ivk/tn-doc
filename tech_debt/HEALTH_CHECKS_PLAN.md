# План Health Checks (liveness/readiness)

## Цель
Добавить эндпоинты здоровья для оркестраторов/мониторов, чтобы корректно перезапускать и балансировать сервис.

## Доработки
1) Liveness
- Проверка жизнеспособности процесса (основные зависимости не проверяются) → быстрый ответ.

2) Readiness
- Проверка готовности: успешная загрузка конфигов, доступность активного канала БД (пинг), наличие шаблонов/каталогов.

3) Интеграция
- `/health/live`, `/health/ready` с тегами и кэшированием.
- Регистрировать в DI, добавить в Startup/Program.

4) Тесты
- Юнит‑тесты провайдера, интеграционные — проверка статусов при падении БД/конфигов.

## Критерии готовности
- Корректные статусы при штатной работе и деградации.
