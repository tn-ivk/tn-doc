# План унификации обработки ошибок API (ProblemDetails)

## Цель
Единый формат ошибок API, понятные статусы и трассировка для клиентов.

## Доработки
1) Middleware/Filters
- Глобальный обработчик исключений → `ProblemDetails` (RFC 7807) с `traceId`.
- Маппинг бизнес‑исключений на статусы: 400/403/404/409/422/500.

2) Контроллеры
- `ProducesResponseType` для успешных/ошибочных ответов.
- Валидационные ошибки → автоматический `ValidationProblemDetails`.

3) Логи
- Логировать исключения один раз на границе (middleware) с уровнем Error и `traceId`.

4) Тесты
- Snapshot‑тесты формата `ProblemDetails` для типовых ошибок.

## Этапы
- Этап 1: middleware и базовый маппинг исключений.
- Этап 2: аннотации контроллеров + тесты.

## Критерии готовности
- Все необработанные исключения возвращают `ProblemDetails`.
- Тесты на JSON‑формат ошибок зелёные.
