# План качества кода (анализаторы, StyleCop/Roslyn)

## Цель
Повысить читаемость и единообразие кода, ловить ошибки на этапе сборки.

## Доработки
1) Анализаторы
- Включить `Microsoft.CodeAnalysis.NetAnalyzers` + `StyleCop.Analyzers` в проектах.
- Настроить `ruleset`/`editorconfig` под конвенции репозитория.

2) CI
- Цель `dotnet build -warnaserror` для критичных правил.
- Отчёт о предупреждениях в CI.

3) Правила
- Именование (PascalCase/camelCase), явные `public` API, запрет `dynamic` без нужды.
- Улучшение логики: ранний возврат, ограничение вложенности, явное освобождение ресурсов.

4) Тесты
- Проверить отсутствие новых предупреждений; baseline для существующих (поэтапно снижать).

## Критерии готовности
- Чистая сборка без новых предупреждений в CI.
