# Конфигурация

## Где лежат конфиги

- **Разработка**: `TN_Doc/Cfg/`
- **Linux-пакет**: `/opt/TN_Doc/Cfg/`
- **Windows-пакет (MSI)**: `C:\ProjectVU\DotNetComponents\TN_Doc\Cfg\` (по умолчанию, настраивается при установке)

Основные пути задаются относительно корня приложения.

## Основные файлы

- `CfgApp.json` — основная конфигурация (устройства, документы, БД, OPC, ELIS).
- `CfgApp.Development.json` — локальные/дев-изменения (опционально).
- `Cfg*.json` — конфигурации конкретных документов.
- `CfgEdit*.json` — конфигурации форм редактирования (если используются).
- `appsettings.json` — настройки ASP.NET Core.
- `nlog.config` — логирование.

## Ключевые разделы `CfgApp.json`

- `Devices[]` — список устройств ИВК.
  - `DBConnectionStrings[]` — подключения к БД (Server, Userid, Password, Database, ConnectionTimeout).
  - `OpcConnectionSettings` — OPC настройки per-device.
  - `UsedSI` — используемые средства измерения (UsedPR, UsedPP, UsedPVL, UsedPVS, SecondSI).
  - `Docs[]` — документы устройства с шаблонами.
    - Наименования паспортов приведены к унифицированному формату «Паспорт качества нефти {стандарт}»:
      - Паспорт качества нефтепродукта
      - Паспорт качества нефти МИ 3532-2015. Приложение 16 (Экспорт)
      - Паспорт качества нефти ГОСТ Р50.2.40. Приложение Ж
      - Паспорт качества нефти ГОСТ Р50.2.40. Приложение И
      - Паспорт качества нефти ТР ЕАЭС
- `ArmOpcConnectionSettings` — глобальные настройки OPC для ARM:
  - `Type`: `DA` или `UA`
  - `DaSettings` — параметры OPC DA (Host, ProgId, StartPrefix, UpdateRate)
  - `UaSettings` — параметры OPC UA (ConfigFilename, UpdateRate)
- `Elis` — параметры интеграции ELIS (Use, OstKey, SiknKey, ClientName, ClientToken).
- `ExportDoc` — настройки экспорта документов (Path).
- `UseSecurityFeatures` — флаг включения функций безопасности (шифрование паролей).
- `DeviceConnectionDiagnostic` — настройки диагностики подключений:
  - `InitialPollSeconds` (1–3600) — начальный интервал опроса
  - `MaxPollSeconds` (60–86400) — максимальный интервал
  - `PollMultiplier` (1.1–10) — множитель увеличения интервала
  - `NetworkFailureThreshold` (1–100) — порог сетевых ошибок
  - `MaxRetryCount` (1–1000) — попытки до перехода в HalfOpen

## Редактирование

Рекомендуемый способ — **Configurator** (`/configurator`), т.к. он валидирует данные и сохраняет `CfgApp.json` безопасно. Ручное редактирование возможно, но делайте резервную копию.

## Проверка после изменений

- Перезапустите приложение/службу.
- Проверьте `/api/status` и логи NLog.

## Миграция конфигурации при обновлении

При обновлении приложения пользовательские настройки из `Cfg/` автоматически мигрируются в новую версию конфигурации с помощью утилиты **cfg-elevator**. Утилита переносит пользовательские значения (адреса серверов, пароли, настройки устройств) из старых конфигов в новые, сохраняя при этом новые свойства целевой версии.

### Алгоритм миграции

1. Перед удалением старых файлов текущая `Cfg/` копируется во временную директорию
2. Устанавливаются новые файлы (включая новые конфиги со значениями по умолчанию)
3. `cfg-elevator migrate` — переносит пользовательские значения из старых конфигов в новые
4. `cfg-elevator fix` — исправляет конфигурацию под новую версию
5. Временная папка и бинарник cfg-elevator удаляются

### Расположение лога миграции

| Платформа | Лог |
|-----------|-----|
| Linux | `/opt/TN_Doc/logs/cfg-elevator.log` |
| Windows | `<INSTALLFOLDER>\logs\cfg-elevator.log` |

Ошибки миграции не прерывают установку — новые конфиги будут содержать значения по умолчанию. Пользовательские настройки можно восстановить вручную из бэкапа.

## Автоматическое бэкапирование

### Linux (.deb)

При установке .deb пакета автоматически создаётся резервная копия текущей конфигурации в `/var/backups/TN_Doc/`. Бэкап включает файлы конфигурации и данные (логи исключаются), что позволяет безопасно откатиться при необходимости.

### Windows (MSI)

При установке/обновлении через MSI автоматически создаётся бэкап в `C:\ProgramData\TN_Doc\backups\` (если директория установки не пуста, исключая `logs/`). После бэкапа директория очищается и устанавливаются новые файлы.
