## Обзор интерфейса

TN_Doc — веб‑приложение ASP.NET Core (Razor Views) c Bootstrap‑гридом и jQuery‑компонентами, с отдельным SPA‑компонентом StatusBar (Vue 3 + SignalR). Главная страница (`Home/Index`) содержит левую панель фильтров и правую панель просмотра документа (PDF в iframe), верхнее меню действий и модальные окна для «Справочников» и «Конфигуратора» (в iframe).

## Палитра и стили

- **Брендовый синий:** `#1b6ec2` (кнопки/hover), `#155a9e` (hover), `#1861ac` (бордеры)
- **Ссылки:** `#0366d6`
- **Сообщения об ошибках/валидатор:** `#dc3545` (фон/бордюр, тултипы)
- **Серые границы/фон:** `#ced4da`, `#e5e5e5`, `#e0e6ef`, `#f5faff`, `#fafbfc`, `#e6f0ff`
- **Текст:** `#495057`, базовый `#222` для списков

Используются стили:
- `wwwroot/css/site.css` — базовые, Bootstrap‑совместимые
- `wwwroot/css/LeftPanel.css`, `wwwroot/css/newstyle.css` — таблицы, кнопки, поля
- `wwwroot/css/menu-dropdown.css` — меню действий
- `wwwroot/css/dictionaries/*` — UI справочников (меню слева, таблицы, элементы выбора)
- `wwwroot/statusbar/status-bar.css` — визуальные состояния индикаторов статуса

## Ключевые графические элементы

- **Кнопки действий:** фон `#1b6ec2`, hover `#155a9e`, disabled `#cccccc`
- **Dropdown меню:** кастомные пункты с hover‑состояниями (сплошной, подсвеченный, градиентный)
- **Индикаторы статуса:** цвета состояний — online (зелёный), offline (красный с миганием), ndv (серый), warning (жёлтый)
- **Валидация форм/полей:** бордер `#dc3545`, мягкий розовый фон, тултипы на `#dc3545`
- **Табличные списки:** заголовки с `#e6f0ff`, границы `#e0e6ef`, hover‑фон `#f5faff`

## Конфигуратор: Design Tokens и Spacing System

Конфигуратор использует собственную систему design tokens для единообразных отступов и анимаций:

```css
.configurator-container {
  --space-1: 0.25rem;   /* 4px — микро-отступы */
  --space-2: 0.5rem;    /* 8px — между элементами */
  --space-3: 0.75rem;   /* 12px — padding компонентов */
  --space-4: 1rem;      /* 16px — gap между секциями */
  --space-5: 1.25rem;   /* 20px — между Panel блоками */
  --space-6: 1.5rem;    /* 24px — крупные разделители */
  --transition-fast: 0.15s ease;
  --transition-normal: 0.2s ease;
}
```

Ключевые UI-элементы конфигуратора:
- **Panel-компоненты** — скруглённые углы (`border-radius: 8px`), полупрозрачный фон, hover-shadow
- **Таблица «Используемые СИ»** — CSS Grid вместо flexbox (grid-template-columns с spacer-ячейками)
- **Список устройств** — hover-эффект со сдвигом padding-left, подсветка выбранного элемента с `border-left: 3px solid`
- **Дерево документов** — компактные отступы узлов (`padding: 0.25rem 0.5rem`)
- **Настройки ЕЛИС** — обёрнуты в Panel-компонент для визуальной группировки
- **Секция диагностики** — InputNumber без кнопок инкремента/декремента, ввод только вручную

## Навигация и окна

## Как воспроизвести и обновить скриншоты

1. Запустите бэкенд TN_Doc локально (Development), чтобы был доступен `http://localhost:5000` или задайте `TN_DOC_BASE_URL`.
2. Установите зависимости в `docs/ui-screenshots`:
   ```bash
   cd docs/ui-screenshots
   npm i
   npx playwright install --with-deps
   npm run screenshots
   ```
3. Скриншоты появятся в `docs/assets/ui/`.

Переменная окружения: `TN_DOC_BASE_URL=http://localhost:38509` (или ваш порт IIS Express / Kestrel).

## Рекомендации по улучшению дизайна

1. **Единый дизайн‑токены:** вынести палитру, размеры шрифтов, радиусы в `:root` CSS‑переменные и использовать во всех CSS.
2. **Консолидация стилей:** объединить дробные стилевые файлы, убрать дубли (`.invalid-cell-content`, цвета кнопок) — снизит рассинхронизацию.
3. **Контраст и доступность:** проверить контраст ссылок `#0366d6` на светлом фоне, увеличить контраст disabled‑элементов; добавить focus‑ring.
4. **Единые состояния элементов:** согласовать hover/active для dropdown (сейчас 3 варианта), выбрать один паттерн.
5. **Сетка и отступы:** унифицировать вертикальные ритмы (8px‑шкала), выровнять отступы в фильтрах слева и панели действий справа.
6. **Иконки:** перейти на современный набор (например, `@tabler/icons` или `bootstrap-icons`) и убрать смешение FA‑версий.
7. **Компоненты таблиц:** применить единый компонент таблицы с заголовком, сортировкой и пустыми состояниями; вынести в JS‑компонент.
8. **Перенос модалок в частичные вьюхи:** вынести большие модальные разметки в отдельные `.cshtml` partials для читаемости, добавить aria‑атрибуты.
9. **Темизация:** добавить светлую/тёмную темы через CSS‑переменные (переключатель в меню), сохранить брендовые оттенки.
10. **StatusBar:** выровнять шрифты/отступы, добавить компактный режим и tooltip‑описания статусов; синхронизировать цвета с основной палитрой.

Ты — опытный UI/UX-дизайнер. 
Твоя задача — адаптировать существующий интерфейс приложения TN_Doc (ASP.NET Core Razor, Bootstrap, jQuery) 
под современный визуальный стиль Material Design 3, сохранив логику и компоновку.

Вот описание текущего интерфейса:
- Левый фильтр с выбором устройства, документа, дат и кнопкой “Получить данные”.
- Таблица с результатами и блок просмотра документа (PDF/iframe) справа.
- Верхняя панель с кнопками: Экспорт, Печать, Отправить, Меню (Справочники / Конфигуратор).
- Нижний статус-бар с индикаторами (online/offline/warning).
- Используются таблицы, модальные окна, dropdown-меню и вкладки.

Основные проблемы дизайна:
- Цветовая палитра устаревшая (смешение серого, синего и розового).
- Низкий контраст, маленькие отступы, неакцентированные кнопки.
- Несогласованные hover-состояния и разные стили таблиц.

Требования к новому дизайну:
1. Применить **Material Design 3 (M3)** с акцентом на читаемость и спокойные тона.
2. Основной цвет — **Blue 600 (#1E88E5)**, hover — **Blue 700 (#1565C0)**.
3. Фон и поверхности — **#FAFAFA** и **#F1F3F4**, текст — **#212121**, вторичный — **#5F6368**.
4. Радиусы — 8 px, отступы по 8/16 px.
5. Кнопки — большие, с контрастом и плавным hover-эффектом.
6. Модальные окна и вкладки — с легкими тенями и яркими активными элементами.
7. Добавить светлую и тёмную темы (Material 3 dynamic scheme).
8. Не менять сетку, маршруты и названия элементов — только визуальный стиль и UI-токены.
9. Подготовить новый набор CSS-переменных или Tailwind-токенов:
    ```
    --md-primary: #1E88E5;
    --md-primary-hover: #1565C0;
    --md-secondary: #43A047;
    --md-error: #E53935;
    --md-warning: #F9A825;
    --md-surface: #FAFAFA;
    --md-surface-variant: #F1F3F4;
    --md-outline: #CFD8DC;
    --md-text: #212121;
    --md-text-secondary: #5F6368;
    ```

Результат:
- Обновлённый макет в стиле Material 3.
- Современные карточки, поля, кнопки, таблицы, вкладки, модальные окна.
- Единый визуальный язык и контрастный, но спокойный интерфейс.
